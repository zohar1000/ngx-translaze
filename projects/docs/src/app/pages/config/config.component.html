TranslazeRootModule is initialized with a config object specifying the different languages and how to handle them.

Here is a config example for english and spanish. {{ mdConfigSample }}
The interface and the explanation for each property is listed below. {{ mdConfigInterface }}

Below is the list of options, mandatory options are marked as bold:

Namedescription
langs the array of languages, each item has the following properties:

code a language code, this can be any string like 'en', 'fr' or 'en-us'.
the language file names should all be in the same pattern which includes the language code (e.g, lang-en.component.ts).
name a display name, this can used in a language dropdown selection
default specify if this language will be the default language.
note that option isUseBrowserDefaultLang below takes precedence over this option.
default: false
complementBy specify a language code that this language will be complemented by.
for example, if a spanish language component file holds a language text having some missing or empty keys, than specifying complementBy: 'en' will have those keys replaced by their counterpart keys from the english language component. in this case ngx-translaze will make sure to load the english language component prior of using the spanish language.
loadLang a function to lazy load the component via import statement, it receives the language code as a parameter.
note that lazy loading of a language component will happen only once, from than on the language component will be be served from the browser's cache.
onLangTextReady a function to be invoked when a language text is ready, it receives the language text as a parameter
onLoadError a function to be invoked in case of an error in ngx-translaze
localStorageKey the key under which ngx-translaze will store its data in the local storage.
default: 'translaze'
isUseBrowserDefaultLang when true, the user's browser language will be the default language.
this option takes precedence even if a default is specified in the 'langs' array option.
default: true
